# K리그 앱 배포 가이드

## 단계 1: 로컬 동작 확인 (지금)

```cmd
cd C:\Users\Lee\kleague-app\app
npm install
npm run dev
```

브라우저 자동으로 `http://localhost:5173` 열림. 다음 확인:

- [ ] **순위 탭**: 12팀 순서대로 1~12위 표시 (서울이 1위)
- [ ] **득점 탭**: 야고 1위 5골부터 20명까지 표시
- [ ] **도움 탭**: 이동경 1위 3개부터 9명까지 표시
- [ ] 하단 탭바 클릭 시 페이지 전환
- [ ] 첫 진입 시 "응원하는 팀 선택" 모달 뜸
- [ ] 팀 선택 후 순위 화면에서 해당 팀 행이 하이라이트됨

문제없으면 단계 2로.

## 단계 2: 크롤러 + GitHub Actions

```cmd
cd C:\Users\Lee\kleague-app

git branch -M main

```

repo는 미리 GitHub에서 만들어둘 것 (이름: `kleague-app`, 공개).

### 2-2. 크롤러 코드 추가

(다음 작업 단계에서 `scraper/scrape.js`와 `.github/workflows/scrape-kleague.yml` 추가 예정)

### 2-3. GitHub Actions 동작 확인

push 후 GitHub repo의 Actions 탭에서 첫 실행이 초록불인지 확인. 매일 KST 01:00에 자동 실행되며, 실패 시에도 시드 데이터로 화면은 정상 표시됨.

## 단계 3: 앱인토스 콘솔 등록 + 빌드

### 3-1. 콘솔에서 K리그 앱 생성

[앱인토스 콘솔](https://apps-in-toss.toss.im) 접속 → 새 앱 생성
- 앱 이름: K리그 순위
- 식별자: `kleague-ranking` (granite.config.ts와 일치)

### 3-2. 광고 ID 발급

콘솔 → 광고 관리 → 신규 광고 생성:
- 배너 광고 1개 → ID 복사
- 전면 광고 1개 → ID 복사

### 3-3. 광고 ID 코드에 반영

`app/src/components/BannerAd.tsx`:
```typescript
const IS_AD_PRODUCTION = true; // false → true
const PROD_BANNER_ID = '여기에_복사한_배너_ID';
```

`app/src/components/InterstitialAd.tsx`:
```typescript
const IS_AD_PRODUCTION = true; // false → true
const PROD_INTERSTITIAL_ID = '여기에_복사한_전면_ID';
```

### 3-4. 빌드

```cmd
cd C:\Users\Lee\kleague-app\app
npx ait build
```

성공 시 출력 끝부분에:
```
*  AIT build completed (kleague-ranking.ait)
•  deploymentId: 019xxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

이 deploymentId를 메모.

### 3-5. 콘솔에 배포 등록

콘솔 → 앱 → 배포 관리 → 신규 배포 → deploymentId 입력 → 등록

### 3-6. 검토 요청

콘솔 → 앱 출시 → "검토 요청하기" 버튼

#### 출시 노트 (예시)
```
K리그1 팀 순위와 득점·도움 개인 기록을 매일 자동 갱신으로 제공해요.
응원 팀을 설정하면 즐겨찾기 카드로 강조 표시돼요.
```

#### 앱 내 기능 (3개 등록)

| 한국어 이름 | 영어 이름 | 이동 URL |
|---|---|---|
| K리그 팀 순위 | K League Standings | `intoss://kleague-ranking/` |
| K리그 득점 순위 | K League Top Scorers | `intoss://kleague-ranking/scorers` |
| K리그 도움 순위 | K League Top Assists | `intoss://kleague-ranking/assists` |

### 3-7. 심사 결과 대기

보통 1~3일. 결과는 콘솔과 등록 이메일로 통지.

## 자주 나는 실수

- 광고 ID를 KBO 앱 거랑 똑같이 사용 → 콘솔에서 K리그 앱별로 따로 발급해야 함
- `IS_AD_PRODUCTION = false` 그대로 빌드 → 테스트 광고가 노출돼서 리젝
- BottomRouter / BrowserRouter 사용 → MemoryRouter여야 함 (이미 적용됨)
- 첫 화면에서 뒤로가기 안 됨 → 앱 종료로 처리해야 함 (BackButtonHandler 적용됨)
- 앱 아이콘이 KBO 거 그대로 → 출시 전 K리그 전용으로 교체

## 자동 갱신 흐름 (단계 2 완료 후)

```
경기 종료 (오후~밤)
  ↓
다음날 KST 01:00 - GitHub Actions 자동 실행
  ↓
scraper/scrape.js → K리그 공식 사이트 → data/*.json 갱신
  ↓
git auto-commit & push (Actions가 자동)
  ↓
jsdelivr CDN (~12시간 캐시) 또는 raw.githubusercontent
  ↓
앱이 fetch → 사용자 화면 갱신
```

캐시 새로고침이 필요하면:
```
https://purge.jsdelivr.net/gh/officealone00/kleague-app@main/data/standings.json
https://purge.jsdelivr.net/gh/officealone00/kleague-app@main/data/scorers.json
https://purge.jsdelivr.net/gh/officealone00/kleague-app@main/data/assists.json
```
